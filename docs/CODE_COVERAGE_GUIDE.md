# Sentinel Code Coverage & Assurance Guide

> **Document ID**: COV-0001  
> **Target Baseline**: .NET 10.0 (Native AOT Compatible)  
> **Last Updated**: 2026-06-25  
> **Compliance Targets**: NIST SP 800-53 · FAPI 2.0 Advanced · FedRAMP High (>80% Global, >90% Critical Paths)

---

## 1. Executive Summary

Sentinel enforces strict Test-Driven Development (TDD) and continuous quality gating. This guide provides authoritative instructions for measuring test coverage, generating visual HTML dashboards via `ReportGenerator`, excluding autogenerated artifacts, and enforcing CI/CD quality thresholds.

---

## 2. Prerequisites

Ensure your local development environment meets the following baseline requirements:

1. **.NET 10.0 SDK** installed (`dotnet --version`).
2. **Coverlet Collector** pre-configured via `Directory.Packages.props` (`coverlet.collector`).
3. **ReportGenerator Global Tool** installed. To install or update to the latest version:

```powershell
dotnet tool update --global dotnet-reportgenerator-globaltool
```

---

## 3. Step-by-Step Local Report Generation

### Step 1: Execute Tests & Collect Cobertura XML

Execute the full test suite across all consolidated test projects and instruct the Coverlet engine to emit `cobertura` XML reports:

```powershell
dotnet test Sentinel.slnx -c Release --collect:"XPlat Code Coverage"
```

*Note: This generates individual `coverage.cobertura.xml` files inside each test project's `TestResults/{guid}/` directory.*

### Step 2: Combine XML Artifacts into an HTML Dashboard

Aggregate all generated Cobertura reports into a unified interactive HTML web portal inside the `coveragereport` folder:

```powershell
reportgenerator -reports:"tests\**\coverage.cobertura.xml" -targetdir:"coveragereport" -reporttypes:Html
```

*(On Linux / macOS hosts, use forward slashes: `-reports:"tests/**/coverage.cobertura.xml"`)*

### Step 3: Launch the Visual Portal

Open the generated web portal in your default browser:

* **Windows (PowerShell)**:
  ```powershell
  Start-Process "coveragereport/index.html"
  ```
* **macOS**:
  ```bash
  open coveragereport/index.html
  ```
* **Linux**:
  ```bash
  xdg-open coveragereport/index.html
  ```

## 4. Optimizing Coverage Metrics (Excluding Autogenerated Artifacts)

By default, MSBuild includes autogenerated OpenAPI schemas (e.g., `Microsoft.AspNetCore.OpenApi.Generated`) and JSON Source Generation contexts (`*JsonContext.cs`). These files artificially inflate complexity and Crap Scores (e.g., CRAP 8930) while diluting business logic scores.

To exclude these non-executable business artifacts, add the following `<PropertyGroup>` to your root `Directory.Build.props` file:

```xml
<PropertyGroup>
  <!-- Exclude Source Generators, OpenAPI transformers, and compiler boilerplate from coverage metrics -->
  <ExcludeByAttribute>GeneratedCodeAttribute,CompilerGeneratedAttribute,JsonSourceGenerationOptionsAttribute</ExcludeByAttribute>
  <Exclude>[*.g.cs]*,[*JsonContext]*,[Microsoft.AspNetCore.OpenApi]*</Exclude>
</PropertyGroup>
```

## 5. CI/CD Quality Gating (GitHub Actions)

In staging and production pipelines, coverage validation is enforced automatically. If code coverage drops below the required security baseline (**80% line coverage**), the pipeline fails immediately.

Snippet from `.github/workflows/security-pipeline.yml`:

```yaml
- name: Execute High-Assurance Test Suite with Coverage Gating
  run: |
    dotnet test Sentinel.slnx \
      --configuration Release \
      --no-restore \
      /p:CollectCoverage=true \
      /p:CoverletOutputFormat=cobertura \
      /p:Threshold=80 \
      /p:ThresholdType=line \
      /p:ThresholdStat=total

- name: Upload Consolidated Coverage to Codecov
  uses: codecov/codecov-action@v4
  with:
    token: ${{ secrets.CODECOV_TOKEN }}
    files: "**/coverage.cobertura.xml"
    fail_ci_if_error: true
```

## 6. Troubleshooting

### Warning: `Unable to find a datacollector with friendly name 'XPlat Code Coverage'`
If you encounter this warning during `dotnet test`:
1. Verify that the test project (`.csproj`) explicitly references `coverlet.collector`:
   ```xml
   <PackageReference Include="coverlet.collector">
     <PrivateAssets>all</PrivateAssets>
     <IncludeAssets>runtime; build; native; contentfiles; analyzers; buildtransitive</IncludeAssets>
   </PackageReference>
   ```
2. Ensure you are executing from the solution root containing `Sentinel.slnx`.

### ReportGenerator: `No report files found`
If `reportgenerator` returns empty results, ensure that test execution completed successfully without MSBuild errors (`CS8805`, `CS0017`), as coverage files are emitted only during successful test host tear-down.
